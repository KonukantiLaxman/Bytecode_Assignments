##Bonus Questions:
1. Can rollback be done after commit?
No rollback cannot be done after commit.
Once commit is used or issued:
1.changes are permanently saved in the database.
2.rollback cannot undo those changes.
**Commit saves changes permanent and rollbacks work only before commit.

2. Difference between rollback and rollback to savepoint.
**rollback:
rollback cancels all uncommitted changes made in the current transactions.
undoes every insert, upfdate and delete since last commit.
Ends the transaction.
Database returns to the last committed states.

**Rollback to savepoint:
ROLLBACK TO SAVEPOINT cancels changes only after a specific savepoint.
Undoes partial changes
Does NOT end the transaction
Changes before the savepoint remain safe

3. Why savepoints are used in real-time applications?
Allow partial rollback without losing entire transaction
1.Prevent data loss in multi-step transactions
2.Improve error handling
3.Used in banking, e-commerce, payroll systems
4.Maintain transaction control
